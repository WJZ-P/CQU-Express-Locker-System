# 项目开发计划

## 1. 引言

### 1.1 编写目的
本文档旨在制定“快递柜综合应用系统”的项目开发计划，明确项目目标、范围、进度安排及人员分工，以确保项目在规定期限（15个工作日）内高质量完成。本文档供项目组成员、指导教师及相关利益相关者阅读。

### 1.2 项目背景
随着电子商务的蓬勃发展，物流配送“最后一公里”的问题日益突出。为了提高物流配送效率，解决用户取件时间冲突及隐私安全问题，智能快递柜应运而生。本项目旨在开发一套集硬件模拟、移动端应用（用户端/快递员端）及Web管理后台于一体的综合应用系统。

### 1.3 定义
- **CQU-Express-Lockers-System**：本项目名称，即“快递柜综合应用系统”。
- **MDesign**：用于分析设计模型的工具。
- **Gitee**：项目代码及文档托管平台。

### 1.4 参考资料
1. 《快递柜综合应用系统》需求说明书
2. 《综合设计要求说明》
3. 《软件工程：理论与实践》

## 2. 项目概述

### 2.1 工作内容
本项目主要工作内容包括：
1.  **硬件产品设计**：设计支持220V电源、电表、摄像头、数字键盘、多规格仓门（带电磁锁/传感器）的智能快递柜硬件方案（软件模拟实现）。
2.  **移动端应用开发**：
    *   **快递员端**：扫码登录、投递快递（选仓）、短信通知。
    *   **用户端**：扫码/码取件、寄存物品、寄件申请。
3.  **Web管理平台开发**：柜体状态监控、用户/快递员管理、数据统计分析。
4.  **文档撰写**：完成项目计划、需求规格、设计规格、测试报告等全套文档。

### 2.2 交付期限
项目开发周期为15个工作日，分为三个迭代阶段（第19周至第21周）。

### 2.3 交付成果
1.  **程序代码**：包含服务端（Spring Boot）、移动端（Android/Uni-app）、Web前端（Vue）及硬件模拟程序。
2.  **项目文档**：
    *   项目开发计划
    *   需求规格说明书
    *   设计规格说明书（体系结构、详细设计、数据库设计）
    *   用户界面设计
    *   软件工程课程设计报告
3.  **设计模型**：MDesign模型文件、Axure/墨刀原型文件。

## 3. 实施计划

### 3.1 过程模型
本项目采用**敏捷开发（Agile）**模式，将开发周期划分为三个迭代（Sprint），每个迭代周期为一周（5个工作日）。

### 3.2 进度安排
| 阶段 | 时间 | 主要任务 | 交付物 |
| :--- | :--- | :--- | :--- |
| **迭代一** | 第19周 | 需求分析、系统建模、数据库设计、UI原型设计、环境搭建 | 需求文档、设计文档、UI原型、基础框架代码 |
| **迭代二** | 第20周 | 核心功能开发（存取件流程）、硬件模拟逻辑实现、移动端基础功能 | 移动端App（Alpha版）、硬件模拟器、中期汇报 |
| **迭代三** | 第21周 | Web后台开发、寄件/寄存功能、系统集成、测试与Bug修复 | 完整系统、测试报告、最终演示 |
| **收尾** | 第22周 | 文档汇总、课程设计报告撰写 | 课程设计报告、项目归档 |

### 3.3 关键问题与对策
-   **硬件模拟**：由于无真实硬件，需开发“硬件模拟器”软件，通过界面模拟柜门开关、传感器状态变化，并与后端通过Socket/HTTP通信。
-   **前后端联调**：制定严格的RESTful API接口文档（Swagger/Apifox），确保前后端并行开发时的一致性。
