# 项目开发计划（迭代版）

## 1. 迭代概述
本项目遵循敏捷开发原则，将15个工作日的开发周期划分为三个核心迭代（Sprint）。每个迭代均包含分析、设计、编码、测试和评审环节。

## 2. 迭代详细计划

## 3. 风险管理
| 风险项 | 可能性 | 影响程度 | 应对措施 |
| :--- | :--- | :--- | :--- |
| 硬件模拟逻辑复杂，状态不一致 | 中 | 高 | 简化模拟逻辑，优先保证“开门-关门-状态更新”核心闭环，使用状态机模式设计。 |
| 移动端兼容性问题 | 低 | 中 | 统一使用Uni-app开发，优先保证Android端运行正常，忽略iOS特定问题。 |
| 进度滞后 | 中 | 高 | 每日站会同步进度，若滞后则砍掉“人脸识别”等非核心功能（加分项）。 |
